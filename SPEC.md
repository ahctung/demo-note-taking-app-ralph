# Technical Specification — Note Taking App

## 1. Project Overview

A full-stack web application that allows authenticated users to create and manage rich-text notes. Notes can optionally be shared publicly via a unique link, accessible to anyone without logging in.

**Learning goals:** Next.js App Router, SQLite persistence, better-auth authentication, TipTap rich text editing, and public content sharing patterns.

---

| Layer | Choice | Notes |

| Package manager | npm | — |

## 3. Feature Requirements

### 3.1 Authentication

- Users can **sign up** with email and password.
- Users can **sign in** with email/password or Google OAuth.
- Users can **sign out**.
- All note management routes are protected; unauthenticated users are redirected to `/login`.
- better-auth handles session management (cookie-based sessions).

### 3.2 Note Management (authenticated users only)

| Action | Description |
|---|---|
| Create | User opens a new blank note in the editor and saves it |
| Read | User sees a list of all their notes; can open any note to read/edit |
| Update | User edits note content and/or title and saves |
| Delete | User deletes a note (with a confirmation step) |

Notes are **private by default** — only the owner can see them.

### 3.3 Note Sharing

- The owner can toggle a note to **"shared"** status.
- On share, a unique `shareToken` (UUID v4) is generated and stored on the note.
- The public URL takes the form `/share/[shareToken]`.
- Anyone (including unauthenticated visitors) can view the note at that URL — **read-only**.
- The owner can **stop sharing** at any time, which invalidates the token (set `isShared = false`). The URL stops working immediately.
- Re-sharing a note **reuses the existing token** (no new URL generated unless explicitly reset).

## 4. Data Model

### 4.1 better-auth managed tables

Managed by better-auth. Do not create manually — run `npx @better-auth/cli generate` to produce the Prisma schema models, then `npx prisma migrate dev`.

better-auth requires **four** core tables:

#### `user` table

| Column | Type | Notes |
|---|---|---|
| id | TEXT (PK) | Generated by better-auth |
| name | TEXT | Display name |
| email | TEXT | Unique |
| emailVerified | INTEGER | Boolean (0/1) |
| image | TEXT | Avatar URL (nullable) |
| createdAt | INTEGER | Unix timestamp (ms) |
| updatedAt | INTEGER | Unix timestamp (ms) |

#### `session` table

| Column | Type | Notes |
|---|---|---|
| id | TEXT (PK) | Generated by better-auth |
| userId | TEXT | FK → user.id (ON DELETE CASCADE) |
| token | TEXT | Unique session token |
| expiresAt | INTEGER | Unix timestamp (ms) |
| ipAddress | TEXT | Nullable |
| userAgent | TEXT | Nullable |
| createdAt | INTEGER | Unix timestamp (ms) |
| updatedAt | INTEGER | Unix timestamp (ms) |

#### `account` table

| Column | Type | Notes |
|---|---|---|
| id | TEXT (PK) | Generated by better-auth |
| userId | TEXT | FK → user.id (ON DELETE CASCADE) |
| accountId | TEXT | Provider-assigned ID (equals userId for credential accounts) |
| providerId | TEXT | e.g. `"google"`, `"credential"` |
| accessToken | TEXT | Nullable |
| refreshToken | TEXT | Nullable |
| accessTokenExpiresAt | INTEGER | Nullable |
| refreshTokenExpiresAt | INTEGER | Nullable |
| scope | TEXT | Nullable |
| idToken | TEXT | Nullable |
| password | TEXT | Hashed password for email/password auth (nullable) |
| createdAt | INTEGER | Unix timestamp (ms) |
| updatedAt | INTEGER | Unix timestamp (ms) |

#### `verification` table

| Column | Type | Notes |
|---|---|---|
| id | TEXT (PK) | Generated by better-auth |
| identifier | TEXT | Identifier for the verification request |
| value | TEXT | Value to be verified |
| expiresAt | INTEGER | Unix timestamp (ms) |
| createdAt | INTEGER | Unix timestamp (ms) |
| updatedAt | INTEGER | Unix timestamp (ms) |

### 4.2 `notes` table

| Column | Type | Constraints | Notes |
|---|---|---|---|
| id | TEXT | PK | UUID v4, generated at insert |
| userId | TEXT | NOT NULL, FK → user.id | Owner of the note |
| title | TEXT | NOT NULL, default `'Untitled'` | Plain text, max 255 chars |
| content | TEXT | NOT NULL, default `''` | TipTap JSON (stringified) |
| isShared | INTEGER | NOT NULL, default `0` | Boolean (0 = private, 1 = shared) |
| shareToken | TEXT | UNIQUE, nullable | UUID v4; generated on first share |
| createdAt | INTEGER | NOT NULL | Unix timestamp (ms) |
| updatedAt | INTEGER | NOT NULL | Unix timestamp (ms) |

> **Content storage:** TipTap's `getJSON()` output is serialised as a JSON string. This preserves the document structure and makes re-hydration via `editor.setContent()` straightforward.

## 5. Application Routes

### 5.1 Route Map

| Route | Access | Purpose |
|---|---|---|
| `/` | Public | Landing / redirect (→ `/notes` if logged in, else `/login`) |
| `/login` | Public (guests only) | Sign in with email/password or Google |
| `/register` | Public (guests only) | Create a new account |
| `/notes` | Protected | List all notes for the logged-in user |
| `/notes/new` | Protected | Create a new note |
| `/notes/[id]` | Protected (owner only) | View and edit a specific note |
| `/share/[shareToken]` | Public | Read-only view of a shared note |
| `/api/auth/[...all]` | Public | better-auth handler (all auth endpoints) |

### 5.2 Route Details

#### `/notes`
- Fetches all notes belonging to the current user, ordered by `updatedAt DESC`.
- Displays a card/list for each note showing: title, creation date, shared badge (if applicable).
- "New Note" button links to `/notes/new`.

#### `/notes/new`
- Renders the TipTap editor with an empty document and an empty title field.
- On save (button or `Cmd/Ctrl+S`): creates the note via a Server Action and redirects to `/notes/[id]`.

#### `/notes/[id]`
- Loads the note (verifying `userId` matches the session).
- Returns 404 if not found or not owned by the current user.
- Renders the TipTap editor pre-populated with the stored content.
- **Toolbar actions:** Save, Delete, Share/Unshare toggle.
- Share toggle calls a Server Action that flips `isShared` (and generates `shareToken` if first share).
- When shared, displays the public URL with a copy-to-clipboard button.

#### `/share/[shareToken]`
- Looks up the note by `shareToken` where `isShared = 1`.
- Returns 404 if not found or not currently shared.
- Renders the note **read-only** (TipTap in `editable: false` mode).
- Shows note title and content only — no user-specific UI.

## 7. TipTap Editor Configuration

### 7.1 Extensions

```ts
import StarterKit from '@tiptap/starter-kit'
import { CodeBlockLowlight } from '@tiptap/extension-code-block-lowlight'
import { createLowlight } from 'lowlight'

const extensions = [
  StarterKit.configure({
    heading: { levels: [1, 2, 3] },
    codeBlock: false,           // replaced by CodeBlockLowlight
    horizontalRule: true,
    bulletList: true,
    bold: true,
    italic: true,
    code: true,                 // inline code
  }),
  CodeBlockLowlight.configure({
    lowlight: createLowlight(),
  }),
]
```

### 7.2 Toolbar Buttons

| Button | TipTap command |
|---|---|
| Bold | `toggleBold()` |
| Italic | `toggleItalic()` |
| Heading 1 | `toggleHeading({ level: 1 })` |
| Heading 2 | `toggleHeading({ level: 2 })` |
| Heading 3 | `toggleHeading({ level: 3 })` |
| Normal text | `setParagraph()` |
| Inline Code | `toggleCode()` |
| Code Block | `toggleCodeBlock()` |
| Bullet List | `toggleBulletList()` |
| Horizontal Rule | `setHorizontalRule()` |

### 7.3 Content persistence

- On save: `editor.getJSON()` → `JSON.stringify()` → stored in `content` column.
- On load: `JSON.parse(content)` → passed to `editor` via `content` prop or `editor.setContent()`.

## 8. Authentication Setup (better-auth)

### 8.1 Server config (`lib/auth.ts`)

```ts
import { betterAuth } from 'better-auth'
import { prismaAdapter } from 'better-auth/adapters/prisma'
import { db } from './db'

export const auth = betterAuth({
  database: prismaAdapter(db, { provider: 'sqlite' }),
  emailAndPassword: { enabled: true },
  socialProviders: {
    google: {
      clientId: process.env.GOOGLE_CLIENT_ID!,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
    },
  },
})
```

### 8.2 Client config (`lib/auth-client.ts`)

```ts
import { createAuthClient } from 'better-auth/react'
export const authClient = createAuthClient()
```

### 8.3 Route protection

Use Next.js middleware (`middleware.ts`) to redirect unauthenticated requests from protected paths to `/login`. Better-auth provides a `getSession()` helper that works in middleware.

### 8.4 Required environment variables

```
BETTER_AUTH_SECRET=          # Random secret (32+ chars)
BETTER_AUTH_URL=             # e.g. http://localhost:3000
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
DATABASE_URL=file:./data/notes.db # Path to SQLite file (Prisma requires the file: prefix)
```

## 9. Server Actions (`actions/notes.ts`)

All actions validate that the current session user owns the target note before mutating.

| Action | Inputs | Behaviour |
|---|---|---|
| `createNote` | `{ title, content }` | Inserts new note row; returns new note id |
| `updateNote` | `{ id, title, content }` | Updates title, content, updatedAt |
| `deleteNote` | `{ id }` | Deletes note row; redirects to `/notes` |
| `toggleShare` | `{ id }` | Flips `isShared`; generates `shareToken` if null; returns updated note |

## 10. UI & Styling Guidelines

- **Layout:** Fixed top navigation bar with the app logo and a "Sign out" button. Main content area is centered with a max width of `4xl` (`56rem`).
- **Notes list:** Responsive grid (1 column mobile, 2 columns tablet+). Each card shows title, last-updated timestamp, and a "Shared" badge if applicable.
- **Editor page:** Full-width editor below a sticky toolbar. Title is an unstyled `<input>` above the editor area. Action buttons (Save, Delete, Share) sit in the top-right of the toolbar.
- **Public share page:** Minimal layout — just the note title, read-only content, and a small "Made with NoteApp" footer. No navigation.
- **Colour palette:** Neutral greys for the shell; a single accent colour (e.g. `indigo-600`) for interactive elements.
- **Responsive:** Mobile-first. The editor should be usable on small screens (toolbar wraps as needed).

## 11. Error Handling & Edge Cases

| Scenario | Handling |
|---|---|
| User visits `/notes/[id]` for another user's note | Return 404 (do not reveal existence) |
| User visits `/share/[token]` for an unshared note | Return 404 |
| Note save fails | Show inline error message; do not navigate away |
| Delete confirmation | Show a modal/dialog; only delete on explicit confirmation |
| Session expires mid-session | better-auth middleware redirects to `/login` |
| Google OAuth — email already registered | better-auth links accounts automatically |

## 12. Project Setup Checklist

1. `npx create-next-app@latest note-app --typescript --tailwind --app --src-dir no`
2. `npm install better-auth @prisma/client`
3. `npm install -D prisma`
4. `npm install @tiptap/react @tiptap/starter-kit @tiptap/extension-code-block-lowlight lowlight`
5. Run `npx prisma init --datasource-provider sqlite` to scaffold `prisma/schema.prisma` and `.env`
6. Configure `lib/db.ts`, `lib/auth.ts`, `lib/auth-client.ts`
7. Run `npx @better-auth/cli generate` to append the four better-auth models into `prisma/schema.prisma`, then add the `Note` model manually
8. Run `npx prisma migrate dev --name init` to apply the schema and generate the Prisma client
9. Add better-auth route handler at `app/api/auth/[...all]/route.ts`
10. Add `middleware.ts` at the project root for route protection
11. Populate `.env.local` with all required environment variables (Prisma reads `DATABASE_URL` from `.env` by default; copy other vars to `.env.local`)
12. Set up Google OAuth credentials in Google Cloud Console; add `http://localhost:3000/api/auth/callback/google` as an authorised redirect URI
